Cursor é um recurso bastante interessante em banco de dados pois permite que seus códigos SQL façam uma varredura
de uma tabela ou consulta linha-por-linha, realizando mais de uma operação se for o caso.

Na maioria das vezes, um simples SELECT exibe na tela essa varredura, trazendo todos os registros de consulta em
questão. A vantagem de usar um cursor é quando, além da exibição dos dados, queremos realizar algumas operações
sobre os registros. Se o volume de operações for grande, fica muito mais fácil, limpo e prático escrever o código
utilizando cursor, do que uma consulta SQL.

Por exemplo: É muito mais vantajoso criar um cursor que faça a análise de cada produto do estoque, conferindo seu
histórico, calculando sua previsão de vendas para o próximo mês, capturando o melhor cliente que já o comprou,
etc, do que criar um select absurdamente grande que talvez não consiga ainda todas as informações de forma simples
